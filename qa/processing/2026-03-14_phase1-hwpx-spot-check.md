# Phase1 HWPX Spot Check

## 대상

- `DOC-POL-005`
- `DOC-POL-007`
- `DOC-POL-009`

## 점검 목적

- HWPX pure text가 실제 태깅 전 작업지로 쓸 수 있는지 확인
- multiline structured block과 marker-only 조각이 본문 품질을 해치지 않는지 확인

## 적용한 보정

- 구조형 블록 내부 줄바꿈은 기본적으로 공백으로 접합
- 다음 줄이 `ㅇ`, `□`, `☞`, `⇒`, `⇨`, `※`, `*`, 원형 숫자 heading으로 시작하면 새 문단으로 분리
- `ㅇ`, `□`, `☞`, `⇒`, `⇨`만 남은 marker-only 블록은 제거

## 결과 요약

- `DOC-POL-005`: `476 -> 485` 문단
- `DOC-POL-007`: `261 -> 261` 문단
- `DOC-POL-009`: `293 -> 292` 문단
- 세 문서 모두 embedded newline `0건`
- 세 문서 모두 marker-only block `0건`

## 문서별 메모

- `DOC-POL-005`
  - `ㅇ 현장의견 ...`
  - `☞ 개선방향 ...`
  - 위 두 줄이 별도 bullet로 분리돼 읽기와 태깅이 쉬워졌다.
  - multiline note도 문장형으로 접합되어 CSV 셀 안 줄바꿈이 사라졌다.

- `DOC-POL-007`
  - note 내부의 줄바꿈이 공백으로 접합되어 long note가 안정적으로 한 문단으로 정리됐다.
  - 구조 자체는 이미 양호했고, 이번 보정은 clean-up 성격이 강하다.

- `DOC-POL-009`
  - 고립 `ㅇ` bullet 1건이 제거됐다.
  - note 내부 줄바꿈도 공백 접합으로 정리됐다.

## 남은 이슈

- `DOC-POL-005`의 `현장의견`, `개선방향`은 아직 띄어쓰기 보정 대상이 아니다.
- `DOC-POL-005`의 `➀`, `➁`, `➂` 계열은 현재 `paragraph`로 남아 있으나, 정보구조상 heading 후보로 볼 여지가 있다.
- `DOC-POL-007`, `DOC-POL-009`의 일부 긴 note는 의미상 2개 이상의 하위 항목을 포함하지만, 현재는 한 note로 유지한다.

## 판단

세 문서 모두 본문 pure text는 태깅 전 단계에서 바로 사용 가능한 수준이다.
현재 HWPX 경로의 핵심 이슈는 구조 손상보다 세부 라벨링 정책에 가깝다.
