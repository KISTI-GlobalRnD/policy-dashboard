# 전략 검토 가이드

이 문서는 `policy_item -> strategy` 수동 검토를 위한 운영 기준이다.

## 입력 파일

- 전체 결정 파일: `qa/ontology/review_queues/policy-item-strategy-review-decisions.csv`
- 정책별 패킷: `qa/ontology/review_packets/strategy_by_policy/*.csv`
- 배치 파일: `qa/ontology/review_packets/strategy_batches/*.csv`
- 배치 인덱스: `qa/ontology/review_packets/strategy-review-batch-index.csv`

실제 편집은 배치 파일 기준으로 진행한다.  
파이프라인 실행 시 배치 편집값이 master decision CSV로 병합된다.

## 상태값

- `pending`: 아직 판단하지 않음
- `reviewed`: 전략 매핑을 확정
- `no_strategy`: 15개 전략 중 어느 하나로 묶지 않음
- `deferred`: 문맥 부족 또는 원문 추가 확인 필요

## 판단 원칙

1. 정책 항목이 특정 전략의 목표물, 산업, 기술군, 제도 지원 대상을 직접 지시하면 `reviewed`
2. 정책 항목이 일반적 연구행정, 연구생태계, 평가제도, 투자체계, 조직운영, 인력지원에 머물면 `no_strategy`
3. 부문이 `인프라·제도` 또는 `인재`라도, 대상이 명시적으로 특정 전략 분야면 전략을 부여할 수 있음
4. 단지 `전략기술`, `혁신`, `육성`, `인프라`, `플랫폼` 같은 일반어만 있으면 전략 근거로 보지 않음
5. `reviewed_secondary_strategy_ids`는 실제로 둘 이상의 전략에 동등하게 걸쳐 있을 때만 사용

## 신뢰도 기입

- `high`: 항목 문장에 전략 대상이 직접 드러남
- `medium`: 문맥상 전략 연계가 분명하지만 직접 표기가 약함
- `low`: 전략 분류를 유지하되 추가 검토 여지가 큼

## 자주 나오는 오분류 패턴

### `STR-002` 과대추천

`연구개발 생태계 혁신방안`, `기초연구 생태계 육성 방안`에서 `전략기술` 표현만으로 `STR-002`가 추천되는 경우가 많다.  
반도체/이차전지/초격차 기술 자체를 다루지 않으면 그대로 채택하지 않는다.

### `STR-004` 과대추천

`인프라`, `플랫폼`, `네트워크`라는 일반어만으로 `차세대 네트워크(6G) 및 지능형 인프라`에 끌리는 경우가 있다.  
6G, AI 네트워크, 오픈랜, 국가 AI 고속도로 등 명시적 맥락이 없으면 보수적으로 본다.

### `STR-001` 과대추천

`AI`가 도구로만 쓰인 경우와 `AI 전략 자체`를 구분해야 한다.  
예를 들어 평가 자동화, 행정 효율화, 일반 연구지원에서 AI를 사용한다는 이유만으로 `STR-001`을 주지 않는다.

## 정책별 빠른 기준

### `POL-010 연구개발 생태계 혁신방안`

- 기본값은 보수적으로 본다.
- 연구행정, 예산심의, 평가개선, 제도개편, 투자체계는 대체로 `no_strategy`
- 특정 전략군의 실증, 사업화, 산업 전개가 직접 나오면 그때만 전략 부여

### `POL-007 기초연구 생태계 육성 방안`

- 기초연구 일반 지원, 블록펀딩, IBS 운영, 연구지원인력은 대체로 `no_strategy`
- `기초연구AI 센터`, `AI 융합 연구자 양성`처럼 AI 자체가 정책 대상이면 `STR-001` 검토

### `POL-012 초혁신경제 15대 프로젝트 추진계획`

- 개별 프로젝트 단위라 전략 직접성이 상대적으로 높음
- 배터리, 바이오, 디지털헬스, 게임/콘텐츠, 양자/초전도처럼 대상이 분명하면 적극 매핑

## 최소 입력 규칙

배치 파일에서 아래 3개만 채워도 파이프라인 반영은 가능하다.

- `decision_status`
- `reviewed_primary_strategy_id`
- `reviewer_name`

`decision_status=no_strategy`일 때는 `reviewed_primary_strategy_id`를 비워둔다.
