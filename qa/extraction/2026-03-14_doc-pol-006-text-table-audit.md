# DOC-POL-006 Text And Table Audit

## 대상

- 문서 ID: `DOC-POL-006`
- 문서명: `초혁신경제 15대 프로젝트 추진계획`

## 점검 범위

- 순수 텍스트 추출
- 정규화 텍스트 품질
- Markdown 표 추출
- 기존 `PyMuPDF find_tables()` 결과와의 차이

## 주요 판단

### 1. 텍스트 경로는 usable

- OCR 없이 순수 텍스트 확보가 가능하다.
- 정규화 후 쪽번호와 marker-only 라인은 제거됐다.
- 우선순위였던 문장형 줄바꿈 병합은 usable 수준까지 올라왔다.
- 다만 일부 문장 내부 공백과 반복 citation은 남아 있다.

예:

- 페이지 17: 중복된 회의 표기가 남는다.
- 페이지 26: `국내중소· 벤처기업`처럼 줄바꿈 지점 공백이 남는다.

### 2. 표는 page chunk만으로는 부족

다음과 같은 이유로 표는 아직 `PyMuPDF4LLM page chunk` 단독 경로를 신뢰하면 안 된다.

- 페이지 26은 본문에 `<의료기기 전주기 시장진출 프로세스 비교>`가 명시되지만 page chunk Markdown 표는 없다.
- 같은 페이지에 기존 `find_tables()` 결과 `TBL-DOC-POL-006-016`, `TBL-DOC-POL-006-017`은 존재한다.

즉 현재 표 전략은 다음이 맞다.

- 텍스트: `PyMuPDF4LLM`
- 표: `PyMuPDF find_tables()` 결과를 병행 검토

### 3. 기존 table extraction에는 false positive가 있다

실제 표가 아니라 레이아웃 또는 제목 조각을 표로 잡은 케이스가 있다.

- 페이지 2: `TBL-DOC-POL-006-001` (`Ⅰ | 추진배경`)
- 페이지 3: `TBL-DOC-POL-006-002` (`Ⅲ | 목표 및 추진`)
- 페이지 10: `TBL-DOC-POL-006-006` (문장 조각 분절)
- 페이지 24: `TBL-DOC-POL-006-013` (`디지털 헬스` 제목 조각)

### 4. 다중 페이지 표는 병합이 필요하다

페이지 36, 37, 38은 하나의 일정표가 나뉘어 내려온 상태다.

- 페이지 36: 연도/반기 헤더 일부
- 페이지 37: 중간 헤더
- 페이지 38: 세부 일정 본문

현재 상태로는 대시보드 테이블로 바로 쓰기 어렵고, `multi-page table merge` 규칙이 필요하다.
