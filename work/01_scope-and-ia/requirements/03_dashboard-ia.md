# 대시보드 정보설계 초안

## 원칙

- 처음부터 복잡한 화면보다 탐색 흐름이 명확해야 한다.
- 모든 집계는 원문 근거로 내려갈 수 있어야 한다.
- 전략, 기술분야, 자원유형 세 축이 어디서나 교차 필터로 동작해야 한다.

## 상위 메뉴 초안

### 1. Overview

- 전체 문서 수
- 전략별 정책항목 분포
- 기술분야 분포
- 자원유형 분포
- 최신 업데이트와 데이터 품질 현황

### 2. 정책별 보기

- 문서 기본 정보
- 정책별 핵심 항목 목록
- 자원유형 비중
- 전략 및 기술분야 매핑
- 원문 근거 링크

### 3. 전략별 보기

- 15개 전략 목록
- 전략별 연결 정책
- 전략별 기술분야 분포
- 전략별 대표 근거

### 4. 기술분야별 보기

- 14개 대분류
- 대분류별 중분류 구성
- 기술분야별 연결 정책과 전략
- 기술분야 개요 문서 연결

### 5. 해외 비교

- 국가별 정책 개요
- 국내 전략과의 대응 관계
- 비교 메모와 근거 문서

### 6. 원문 근거 보기

- 문서/페이지 기반 탐색
- 텍스트 블록
- 표
- 그림
- 추출 신뢰도와 검토 상태

## 공통 필터 초안

- 문서
- 기관
- 날짜
- 전략
- 기술 대분류
- 기술 중분류
- 자원유형
- 국내/해외

## drill-down 흐름

1. 전략 또는 기술분야 선택
2. 연결된 정책항목 확인
3. 정책항목별 근거 블록/표/그림 확인
4. 원문 페이지로 이동

## 이번 단계 산출물

- 메뉴 구조 확정
- 필터 구조 확정
- 화면별 핵심 KPI 확정
- 상세 화면의 근거 연결 방식 확정
