# 매핑 규칙 초안 v0.1

## 목적

추출된 근거와 정책항목을 전략, 기술분야, 자원유형에 일관되게 연결한다.

## 매핑 우선순위

1. 원문 명시 표현
2. 재구성 초안의 직접 대응 관계
3. 기술분야 기준표의 표준 용어
4. 동의어/약어 사전
5. 검토자 판단

## 자원유형 매핑 규칙

### `technology`

다음과 같은 성격이면 `technology`로 본다.

- 알고리즘, 모델, 공정, 부품, 장비, 플랫폼 기술
- 실증 대상 핵심 기술
- 성능 목표가 기술 지표로 제시되는 항목

예시 키워드:
- 개발
- 고도화
- 확보
- 실증
- 상용화 기술
- 핵심기술
- 원천기술

### `infrastructure_policy`

다음과 같은 성격이면 `infrastructure_policy`로 본다.

- 제도 개편
- 예산, 펀드, 세제
- 테스트베드, 데이터센터, 인프라 구축
- 규제 특례, 거버넌스, 협의체

예시 키워드:
- 제도
- 지원체계
- 인프라
- 센터
- 클러스터
- 특구
- 펀드
- 세제

### `talent`

다음과 같은 성격이면 `talent`로 본다.

- 인력 양성
- 연구자 확보
- 교육과정
- 해외 인재 유치
- Post-Doc, 석박사, 전문인력

예시 키워드:
- 인재
- 인력
- 양성
- 교육
- 교수
- 연구자
- 석박사
- 채용

## 전략 매핑 규칙

- 전략 매핑의 1차 기준은 `정책기술분야별 재구성(안)`의 15개 전략 축이다.
- 하나의 정책항목이 여러 전략에 걸치면 복수 매핑을 허용한다.
- 다만 대표 전략 1개는 반드시 지정한다.
- 대표 전략을 정할 수 없으면 `review_required` 상태로 남긴다.

## 기술분야 매핑 규칙

- 기술 대분류와 중분류는 엑셀 기준표를 우선한다.
- 문서 표현이 기준표와 다를 경우 normalized label을 추가로 둔다.
- 중분류를 특정할 수 없으면 대분류까지만 매핑한다.
- 대분류조차 특정되지 않으면 `unmapped` 상태로 남기고 검토 대상으로 보낸다.

## 근거 연결 규칙

- `PolicyItem`은 최소 1개의 `EvidenceUnit` 또는 `TableUnit` 또는 `FigureUnit`에 연결되어야 한다.
- 근거 없는 정책항목은 임시 항목으로만 저장하고 대시보드 공개 대상에서 제외한다.
- 표/그림 기반 항목은 본문 블록보다 표/그림 객체를 우선 근거로 삼는다.

## 신뢰도와 검토 상태

### confidence

- `high`: 원문 명시 표현과 기준표가 일치
- `medium`: 동의어 또는 문맥 추론이 필요
- `low`: 사람이 재검토해야 함

### review_status

- `auto_mapped`
- `review_required`
- `reviewed`
- `rejected`

## 예외 처리

- 하나의 항목이 기술과 제도 성격을 동시에 가지면 대표 자원유형과 보조 자원유형을 분리 저장한다.
- 기관명, 부처명, 전략명 약어는 통제어휘 사전에서 먼저 정규화한다.
- 해외 문헌은 국내 전략 기준에 직접 맞지 않으면 `benchmark_only` 태그를 둔다.
