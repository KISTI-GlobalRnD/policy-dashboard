# 실행계획 v0.1

## 설계 원칙

- 원본 데이터와 가공 데이터를 분리한다.
- 추출과 해석을 분리한다. 먼저 구조를 뽑고, 그 다음 분류와 매핑을 한다.
- 모든 산출물은 원문 파일명, 페이지, 블록 단위 근거를 남긴다.
- 표와 그림은 단순 OCR 텍스트가 아니라 별도 구조 객체로 다룬다.
- 온톨로지는 처음부터 무겁게 가지 말고, 최소 데이터모델로 시작한 뒤 확장한다.

## 0. 범위 확정 및 정보설계

목표:
- 어떤 문서를 포함할지, 어떤 질문에 답할지, 어떤 필터로 볼지를 먼저 고정한다.

주요 작업:
- 포함/제외 문서 확정
- 문서 등록부 작성
- 핵심 사용자 질문 정의
- 대시보드 상위 메뉴 정의
- 필수 필터 정의

산출물:
- 범위 정의서
- 문서 등록부
- 질문 목록
- 정보설계 초안
- 의사결정 로그

종료 조건:
- 분석 대상 문서 목록이 확정됨
- 문서별 소스 경로, 형식, 역할, 포함 상태가 정리됨
- `전략`, `기술분야`, `자원유형`의 기준 체계가 확정됨
- 대시보드의 기본 보기 단위가 정해짐

작업 위치:
- `work/01_scope-and-ia/`

## 1. 최소 데이터모델 및 온톨로지 논리설계

목표:
- 추출과 가공에 앞서 어떤 개체와 관계를 남길지 먼저 고정한다.

주요 작업:
- 핵심 엔터티 정의
- 관계 정의
- ID 체계 정의
- 통제어휘 정의
- provenance 규칙 정의
- 추출 산출물의 공통 키 체계 정의

핵심 엔터티 예시:
- 문서
- 정책항목
- 전략
- 기술분야
- 자원유형
- 근거 블록
- 표
- 그림
- 기관

종료 조건:
- 최소 데이터모델이 문서화됨
- 추출 결과와 가공 테이블이 같은 키 체계를 사용함

작업 위치:
- `docs/03_minimum-data-model.md`
- `work/04_ontology/schemas/`

## 2. 샘플 문서 구조화 추출 파일럿

목표:
- 전체 문서에 들어가기 전에 형식별 추출 방식을 검증한다.

주요 작업:
- 골든 샘플 문서 선정
- PDF/HWP/HWPX/XLSX별 시범 추출
- 표/그림 처리 기준 검증
- 추출 누락과 오류 유형 기록

산출물:
- 샘플 추출 결과
- 추출 이슈 로그
- 추출 명세 보정본

종료 조건:
- 최소 1개 PDF, 1개 HWP/HWPX, 1개 XLSX에 대해 구조화 추출 방식이 검증됨
- 단계별 QA 체크리스트가 통과 기준을 갖춤

작업 위치:
- `work/02_structured-extraction/`
- `qa/extraction/`

## 3. 전체 문서 구조화 추출

목표:
- 원문을 재사용 가능한 구조화 단위로 분해한다.

주요 작업:
- PDF/HWP/HWPX/XLSX 파일별 추출 파이프라인 정의
- 본문 블록 추출
- 표 구조 추출
- 그림/도식/캡션 추출
- 페이지, 좌표, 추출 신뢰도 저장

필수 저장 단위:
- 텍스트 블록
- 표 JSON 또는 CSV
- 그림 설명 메타데이터
- 레이아웃 메타데이터
- 문서별 매니페스트

종료 조건:
- 문서별로 `텍스트`, `표`, `그림`, `페이지 근거`가 분리 저장됨
- 추출 누락 문서와 품질 이슈가 기록됨
- 파일럿에서 확정한 명세가 전체 문서에 일관되게 적용됨

작업 위치:
- `work/02_structured-extraction/`

## 4. 데이터 정규화와 매핑

목표:
- 추출된 구조 데이터를 공통 기준으로 정리한다.

주요 작업:
- 문서명, 날짜, 부처명 표준화
- 중복 제거
- 정책 항목 단위 분할
- `기술 / 인프라·제도 / 인재` 분류
- `15개 전략`, `기술 대분류`, `기술 중분류` 매핑
- 대시보드용 집계 테이블 생성

산출물:
- 정규화 테이블
- 매핑 테이블
- 대시보드용 마트

종료 조건:
- 문서 간 비교가 가능한 공통 스키마가 완성됨
- 주요 분류 기준이 일관되게 적용됨
- 매핑 규칙과 예외 처리 규칙이 문서화됨

작업 위치:
- `work/03_processing/`

## 5. 온톨로지 구현

목표:
- 설계된 모델을 실제 저장 구조로 구현한다.

권장 접근:
- 1차 구현은 관계형 테이블 또는 SQLite/Parquet 중심으로 진행
- 2차 필요 시 RDF/OWL 또는 그래프 형태로 내보내기

주요 작업:
- 엔터티 테이블 생성
- 매핑 테이블 생성
- 인스턴스 적재
- 버전 관리 규칙 정의

종료 조건:
- 엔터티와 관계가 실제 데이터 파일 또는 DB로 구현됨
- 대시보드가 참조할 수 있는 안정된 데이터 계약이 생김
- 관계형 구현과 추후 그래프 확장 포인트가 분리되어 있음

작업 위치:
- `work/04_ontology/`

## 6. 대시보드 상세 설계와 연결

목표:
- 확정된 데이터 구조를 기준으로 화면과 상호작용을 설계한다.

주요 작업:
- 화면 요구사항 정의
- 와이어프레임 작성
- 데이터 계약 정의
- 필터, 검색, drill-down 설계
- 근거 문장/표/그림 연결 방식 정의

권장 화면:
- 정책별 보기
- 전략별 보기
- 기술분야별 보기
- 해외 비교
- 원문 근거 보기

종료 조건:
- 화면별 데이터 입력 규격이 정해짐
- 필요한 API 또는 파일 입력 형식이 확정됨

작업 위치:
- `work/05_dashboard/`
- `work/06_integration/`
